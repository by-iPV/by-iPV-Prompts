# /prompts/trc

## 1. Propósito
Almacenar generadores y validadores TRC para contexto conceptual portable.

## 2. Qué archivos contiene
- `README.md`
- `.gitkeep`
- (plantillas TRC)

## 3. Cuándo usarlo
Para construir, revisar y portar contexto conceptual entre sesiones y entornos.

## 4. Qué NO debe hacerse
No convertir TRC en instrucciones de runtime técnico directo.

## 5. Fuente primaria
ChatGPT.

## 6. Fuente secundaria
Codex.

## 7. Relación con MA-AE, N+1, HS v0.9.9, ECOSISTEMA MA-AE BD / HS / MM / PDGD, SDLC, Prompt Registry
TRC mantiene alineación MA-AE, facilita migración N+1 y preserva continuidad conceptual desde HS v0.9.9 hacia ECOSISTEMA MA-AE BD / HS / MM / PDGD dentro del SDLC documental.

## 8. Riesgos si se usa incorrectamente
Confundir contexto conceptual con ejecución técnica puede degradar gobernanza.

## 9. Compatibilidad
- ChatGPT: alta
- Codex Cloud: media
- VSCode: indirecta

## 10. Restricciones operacionales
Uso documental portable; sin impacto en runtime ni UI.
